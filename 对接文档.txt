================================================================================
           呆呆网络 · 生图对接文档（官方 OpenAI + 代理池）
================================================================================

推荐方案：云托管直连 https://api.openai.com，出站经 Webshare 等代理池。
（不再依赖 https://openai.dai520.cn Cloudflare 中转。）


--------------------------------------------------------------------------------
一、云托管环境变量
--------------------------------------------------------------------------------

必填：

  DAIDAI_IMAGE_BASE_URL=https://api.openai.com
  DAIDAI_IMAGE_KEY=sk-你的OpenAI密钥

不要加 /v1。Node 会自己拼路径，例如：
  POST https://api.openai.com/v1/images/generations

选填：

  DAIDAI_IMAGE_MODEL=gpt-image-2          # 默认即此模型
  DAIDAI_PROXY_TRIES=3                    # 代理失败换线次数，默认 3
  ADMIN_PASSWORD=...
  DAIDAI_API_BASE=https://你的云托管域名   # 小程序对接域名，不要末尾 /

关闭（不要再开）：

  DAIDAI_IMAGE_PROXY_ASYNC                # 删掉或不要设为 1


--------------------------------------------------------------------------------
二、代理池（Webshare 等）
--------------------------------------------------------------------------------

格式（每行一条）：

  IP:端口:用户名:密码

方式 A：无需粘贴——仓库已内置 config/proxies.builtin.txt，部署后自动加载
方式 B：管理后台 → 运维配置 → 出站代理池 → 覆盖保存
方式 C：持久盘写入 /app/data/proxies.txt（优先于内置）

后台同页可查看「服务器出口 IP」，仅当 Webshare 要求 IP 白名单时再填。
用「用户名+密码」连代理时，一般不用加白名单。


--------------------------------------------------------------------------------
三、呆呆实际请求（内部，用户侧看不到 OpenAI）
--------------------------------------------------------------------------------

文生图：

  POST {DAIDAI_IMAGE_BASE_URL}/v1/images/generations
  Authorization: Bearer {DAIDAI_IMAGE_KEY}
  Content-Type: application/json

  {
    "model": "gpt-image-2",
    "prompt": "...",
    "size": "1024x1024",
    "n": 1
  }

成功：上游返回 data[0].b64_json 或 data[0].url
失败：上游 error 对象；后端会记入错误日志


探测（不扣生图费）：

  GET {DAIDAI_IMAGE_BASE_URL}/v1/models
  Authorization: Bearer {DAIDAI_IMAGE_KEY}


--------------------------------------------------------------------------------
四、上线自检
--------------------------------------------------------------------------------

1. 部署最新代码
2. 环境变量按上面改好并重启/重新发布
3. 后台粘贴代理池，确认显示「已加载 N 条」
4. 连通探测 → 探测呆呆 Image（应能拉到模型列表）
5. 网页端或小程序试一句「帮我画一张…」


--------------------------------------------------------------------------------
五、和旧中转对照
--------------------------------------------------------------------------------

| 项目           | 旧方案（已弃用优先）           | 现方案（推荐）                    |
|----------------|--------------------------------|-----------------------------------|
| BASE_URL       | https://openai.dai520.cn       | https://api.openai.com            |
| 出站           | Cloudflare Worker              | Webshare 代理池                   |
| 异步           | DAIDAI_IMAGE_PROXY_ASYNC=1     | 不要开                            |
| 密钥           | 同一把 OpenAI sk-              | 同一把 OpenAI sk-                 |


有问题先看管理后台「错误日志」与「连通探测」。
